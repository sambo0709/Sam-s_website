Quaver — Mood to Music

Quaver is a full-stack web application that translates how you're feeling into a curated Spotify playlist. Users pick a mood, set how many songs they want, or hit "Random Mood" to let the app decide — then Quaver queries the Spotify API and returns matching tracks. You can also search songs, artists, or albums directly. The app is deployed live on Render with JWT-authenticated user accounts and persistent MongoDB storage.

Technologies Used

  • Node.js + Express.js — server framework and routing
  • MongoDB — database for user accounts and session data
  • JWT (JSON Web Tokens) — stateless authentication
  • bcryptjs — secure password hashing
  • Spotify API — music data, search, and playlist generation
  • Vanilla HTML, CSS, JavaScript — no framework frontend
  • CSS custom properties — dark/light theme support
  • Google Fonts (Quicksand, Space Grotesk) — typography
  • dotenv — environment variable configuration
  • Render — cloud deployment

Features

  • Mood selection UI that maps emotional states to Spotify search parameters
  • Live Spotify API integration returning real track data and playback previews
  • JWT-based user authentication with protected routes
  • bcrypt password hashing for secure credential storage
  • User account creation and login flow
  • Dark and light theme toggle with CSS custom properties
  • Deployed and accessible live on Render

Technical Architecture

The backend is structured as a REST API with Express.js handling routing and middleware. Authentication uses JWT tokens issued on login and verified on protected endpoints, with passwords hashed using bcrypt before storage. The Spotify API is queried server-side using the Client Credentials flow, keeping API secrets out of the browser. The frontend communicates with the backend via fetch calls and dynamically renders track results without a page reload.

What I Learned

This project deepened my understanding of secure authentication patterns — specifically how JWT and bcrypt work together to keep user data safe without server-side sessions. Integrating the Spotify API taught me how to manage OAuth credentials server-side, handle rate limits, and shape raw API responses into something meaningful for the UI. Deploying on Render also gave me hands-on experience with environment configuration and production build considerations.
